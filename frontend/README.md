## 前端统一规范（表格/表单/交互）

### 表格列表（推荐抽象 BaseTable 组件）
- 列配置驱动，保留筛选区插槽。
- 分页统一 `page/per_page`，底部统一分页组件。
- 多选 + 批量操作：左侧勾选，操作按钮禁用态，执行前二次确认，成功后刷新并清空选中。
- 导出：导出当前筛选结果，操作中显示 loading，完成后提示并自动下载。
- 空/异常：空态统一提示，接口失败 toast 不清空旧数据。
- 筛选保存：支持重置/保存当前筛选条件（localStorage），进入页面优先加载上次筛选；查询参数透传导出/批量接口。

### 表单（推荐抽象 BaseForm 组件）
- 必填项标 *，校验错误展示统一。
- 提交时禁用按钮并显示 loading，失败提示原因。
- 上传（图片/附件）统一封装：大小/格式校验，上传中 loading，失败提示。

### 通用交互
- 权限控制：按钮/菜单基于角色+权限数组控制显隐，不渲染无权限操作。
- 二次确认：危险/批量操作需确认弹窗。
- 通知入口：顶栏统一通知/消息入口（已在布局加入）。
- 环境提示：顶栏展示当前环境（生产/测试）。

### 计划中的公用组件（建议放置 `src/components`）
- `BaseTable`：列配置、筛选插槽、多选、批量操作、分页、导出、空态/loading。
- `BaseForm`：表单布局、校验、提交/重置、禁用态、上传封装。
- `FilterBar`：关键词/下拉/时间范围，提供“保存筛选/重置”回调。
- `ConfirmAction`：带确认的操作按钮，支持批量。
- `NoticeBadge`：全局通知入口（与顶栏 badge 保持一致）。
